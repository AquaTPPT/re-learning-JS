Loops and Arrays

A couple of notes regarding the code:

In the function sumInput(), there are 2 main ways of writing the code for the several different uses: Through an if... else if... else statement or with the OR ( | | ) Brackets.

At first, I thought about writing everything into the script and then comment the least efficient one, but that would be a little confusing to read after a while. Therefore, in order to note myself down in the future, I will write it in the HTML itself.

Using the first method, it could be written like so:

if ( value == null ) {

break

}

else if (!isFinite(value)) {

numbers.pop()
break

}

else if ( value == '' ) {

numbers.pop()
break

}

This is, however, very inefficient.

A better way of writing this would be like so:

if (value == null || !isFinite(value) || value == '') {

break

}

But if you check the code, you'll see that it's written in yet another way. That is because in both of these ways, the array will still contain the final value, since no matter what value it is, it'll still push the value. Now, with the way it is written, if the values turn into strings or anything other than a number, it'll delete that value before continuing. the difference in power that this used is completely negligible.

There is yet another thing though: In the command "sort()", there are also 2 ways of making this one bit:
After checking Stack Overflow, it was mentioned that you could make it with both the commands "sort" and "reverse, since "sort" sorts the numbers in ascending order, and "reverse" inverts the array order.

However, this is NOT how the exercise solution does.

instead, it shows the following:

let arr = [5, 2, 1, -10, 8] ;

arr.sort((a , b) => b - a);

alert( arr ) ;

This is preferable, since it uses one less command than the previous way.
